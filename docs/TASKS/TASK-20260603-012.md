## Intencion funcional

SARA debe poder crear areas simples y asociar informacion existente a ellas.

Ejemplo create:

```text
crear area salud
```

Respuesta esperada:

```text
Area creada: salud
```

Ejemplo list:

```text
que areas tengo
```

Respuesta esperada:

```text
Estas son tus areas activas:
1. salud
```

Ejemplo archive:

```text
archivar area salud
```

Respuesta esperada:

```text
Area archivada: salud
```

Ejemplo assign desde foco:

```text
asociar esa tarea al area salud
```

Respuesta esperada:

```text
Tarea asociada al area salud: <titulo>
```

## Arquitectura esperada

### 1. DB

Crear migracion nueva solo para objetos `sara_`.

Tabla autorizada:

```text
sara_areas
```

Campos minimos segun `docs/ENTITY_CATALOG.md`:

- `id uuid primary key`
- `schema_version text not null`
- `name text not null`
- `slug text not null`
- `description text`
- `status text not null`
- `created_at timestamptz not null`
- `updated_at timestamptz not null`

Restricciones:

- `slug` unico.
- `status` solo `active`, `paused`, `archived`.
- `name` y `slug` no vacios.
- RLS habilitado.
- anon/authenticated sin acceso.
- service_role autorizado.

Eventos en `sara_events`:

- `area_created`
- `area_archived`
- `note_area_assigned`
- `task_area_assigned`

RPCs requeridas:

- `sara_create_area`
- `sara_archive_area`
- `sara_assign_note_area`
- `sara_assign_task_area`

Reglas:

- `sara_create_area` crea area activa y emite `area_created`.
- Si el slug ya existe activo/pausado, debe fallar sin duplicar.
- `sara_archive_area` archiva por `areaId` o `slug` y emite `area_archived`.
- `sara_assign_note_area` actualiza `sara_notes.area_id` y emite `note_area_assigned`.
- `sara_assign_task_area` actualiza `sara_tasks.area_id` y emite `task_area_assigned`.
- No crear tablas puente en este MVP.
- No modificar registros archivados salvo `archive`.

### 4. Parser deterministico

Crear parser simple, por ejemplo:

```text
src/modules/areas/areasParser.ts
```

Debe soportar:

- `crear area salud`
- `nueva area trabajo`
- `que areas tengo`
- `listar areas`
- `archivar area salud`
- `asociar esa tarea al area salud`
- `asignar ultima tarea a salud`
- `asociar esa nota al area aprendizaje`

Extracciones MVP:

- `name`
- `slug`
- `description`
- `status`
- `areaSlug`
- `entityType`: `note` o `task`
- `entityId`
- `position`
- `titleMatch`

Reglas:

- Generar slug deterministico ASCII-safe desde `name` en codigo o SQL.
- Si create no trae name: `missingData = ["areaName"]`.
- Si archive no trae area identificable: `missingData = ["area"]`.
- Si assign no trae area: `missingData = ["area"]`.
- Si assign no trae entidad resoluble: `missingData = ["entity"]`.
- No usar LLM para inventar area ni entidad.

### 6. Router, executor y composer

Registrar acciones:

- `areas.create`
- `areas.list`
- `areas.archive`
- `areas.assign-note`
- `areas.assign-task`

`actionExecutor` debe despachar a handlers reales y bloquear si:

- `intentConfidence < 0.75`
- `missingData` no esta vacio
- faltan entidades requeridas

`responseComposer` debe responder:

- Crear: `Area creada: <name>`
- Listar con resultados: `Estas son tus areas activas:\n1. <name>`
- Listar sin resultados: `No encontre areas activas.`
- Archivar: `Area archivada: <name>`
- Asignar nota: `Nota asociada al area <areaName>.`
- Asignar tarea: `Tarea asociada al area <areaName>: <title>`

Nunca confirmar create/archive/assign sin `areaId` y `eventId`. Para assign tambien debe existir `entityId`.

### 7. Integracion con session-context

Despues de acciones ejecutadas:

`areas.create`:

- foco: `focusedEntityType = "area"`
- `focusedEntityId = areaId`
- `activeModule = "areas"`
- `activeFlow = "area_created"`
- `context.lastAreaName = name`
- `context.lastAreaSlug = slug`

`areas.list`:

- si hay una unica area activa, enfocar esa area.
- si hay varias, guardar `context.lastAreaList` con posiciones, ids, names y slugs.

`areas.archive`:

- limpiar foco si el foco estaba en esa area.

`areas.assign-note` y `areas.assign-task`:

- mantener foco en la entidad asignada y actualizar `context.lastAreaName` / `context.lastAreaSlug`.

Resolucion de referencias:

- Para `asociar esa tarea al area X`, usar `sessionContext.focusedEntityType = task`.
- Para `asignar ultima tarea a X`, usar foco o `context.lastTaskList` si tiene una unica tarea segura.
- Para `asociar esa nota al area X`, usar `sessionContext.focusedEntityType = note`.
- Si hay ambiguedad, retornar missingData y no ejecutar.
