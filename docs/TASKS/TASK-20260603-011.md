## Arquitectura esperada

### 1. DB

Crear migracion nueva solo para objetos `sara_`.

Tabla autorizada:

```text
sara_daily_log
```

Campos minimos segun `docs/ENTITY_CATALOG.md`:

- `id uuid primary key`
- `schema_version text not null`
- `date date not null`
- `wake_energy integer`
- `sleep_hours numeric`
- `morning_intention text`
- `evening_review text`
- `mood text`
- `notes jsonb not null`
- `trace_id uuid`
- `created_at timestamptz not null`
- `updated_at timestamptz not null`

Restricciones:

- unique por `date`.
- `wake_energy` entre 1 y 10 si existe.
- `sleep_hours` >= 0 si existe.
- `notes` default `[]::jsonb`.
- RLS habilitado.
- anon/authenticated sin acceso.
- service_role autorizado.

Eventos en `sara_events`:

- `daily_log_created`
- `daily_log_morning_updated`
- `daily_log_evening_updated`

RPCs requeridas:

- `sara_upsert_daily_log_morning`
- `sara_upsert_daily_log_evening`

Regla de eventos:

- Si el registro no existia, emitir `daily_log_created`.
- Si existia y se actualiza por morning, emitir `daily_log_morning_updated`.
- Si existia y se actualiza por evening, emitir `daily_log_evening_updated`.

No crear tablas adicionales.
