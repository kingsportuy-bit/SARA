## Intencion funcional

SARA debe poder registrar objetivos simples de mediano/largo plazo y vincular tareas existentes a esos objetivos.

Ejemplo create:

```text
crear objetivo mejorar mi energia area salud
```

Respuesta esperada:

```text
Objetivo creado: mejorar mi energia
```

Ejemplo list:

```text
que objetivos tengo
```

Respuesta esperada:

```text
Estos son tus objetivos activos:
1. mejorar mi energia
```

Ejemplo achieve:

```text
marcar objetivo mejorar mi energia como logrado
```

Respuesta esperada:

```text
Objetivo logrado: mejorar mi energia
```

Ejemplo assign desde foco:

```text
asociar esa tarea al objetivo mejorar mi energia
```

Respuesta esperada:

```text
Tarea asociada al objetivo mejorar mi energia: <titulo>
```

## Arquitectura esperada

### 1. DB

Crear migracion nueva solo para objetos `sara_`.

Tabla autorizada:

```text
sara_objectives
```

Campos minimos:

- `id uuid primary key`
- `schema_version text not null`
- `title text not null`
- `slug text not null`
- `description text`
- `area_id uuid`
- `status text not null`
- `target_date date`
- `success_criteria jsonb not null`
- `trace_id uuid`
- `created_at timestamptz not null`
- `updated_at timestamptz not null`
- `achieved_at timestamptz`
- `archived_at timestamptz`

Extender tabla existente:

```text
sara_tasks.objective_id uuid null
```

Restricciones:

- `slug` unico.
- `title` y `slug` no vacios.
- `status` solo `active`, `achieved`, `archived`.
- `success_criteria` default `[]::jsonb`.
- `achieved_at` solo existe cuando `status = achieved`.
- `archived_at` solo existe cuando `status = archived`.
- Si `area_id` existe, debe referenciar un area activa.
- RLS habilitado en `sara_objectives`.
- anon/authenticated sin acceso.
- service_role autorizado.

Eventos en `sara_events`:

- `objective_created`
- `objective_achieved`
- `objective_archived`
- `task_objective_assigned`

RPCs requeridas:

- `sara_create_objective`
- `sara_achieve_objective`
- `sara_archive_objective`
- `sara_assign_task_objective`

Reglas:

- `sara_create_objective` crea objetivo activo y emite `objective_created`.
- Si el slug ya existe activo, debe fallar sin duplicar.
- `sara_achieve_objective` marca logrado por `objectiveId` o `slug` y emite `objective_achieved`.
- `sara_archive_objective` archiva por `objectiveId` o `slug` y emite `objective_archived`.
- `sara_assign_task_objective` actualiza `sara_tasks.objective_id` y emite `task_objective_assigned`.
- No crear `sara_plans`.
- No crear tablas puente en este MVP.
- No generar tareas automaticamente.

- `que objetivos tengo`

Extracciones MVP:

- `title`
- `slug`
- `description`
- `areaSlug`
- `targetDate` solo si viene como `para YYYY-MM-DD`
- `successCriteria` como array de textos solo si viene despues de `criterio:` o `criterios:`

- `taskId`

Reglas:

- Si assign no trae objetivo: `missingData = ["objective"]`.
- Si assign no trae tarea resoluble: `missingData = ["task"]`.
- No usar LLM para inventar objetivo, area, fecha ni criterios.

- `objectives.create`
- `objectives.list`

`responseComposer` debe responder:

- Listar sin resultados: `No encontre objetivos activos.`
- Lograr: `Objetivo logrado: <title>`
- Archivar: `Objetivo archivado: <title>`
- Asignar tarea: `Tarea asociada al objetivo <objectiveTitle>: <taskTitle>`

Nunca confirmar create/achieve/archive/assign sin `objectiveId` y `eventId`. Para assign tambien debe existir `taskId`.

### 7. Integracion con session-context

Despues de acciones ejecutadas:

`objectives.create`:

- foco: `focusedEntityType = "objective"`
- `focusedEntityId = objectiveId`
- `activeModule = "objectives"`
- `activeFlow = "objective_created"`
- `context.lastObjectiveTitle = title`
- `context.lastObjectiveSlug = slug`

`objectives.list`:

- si hay un unico objetivo activo, enfocar ese objetivo.
- si hay varios, guardar `context.lastObjectiveList` con posiciones, ids, titles y slugs.

`objectives.achieve` y `objectives.archive`:

- limpiar foco si el foco estaba en ese objetivo.

`objectives.assign-task`:

- mantener foco en la tarea asignada.
- actualizar `context.lastObjectiveTitle` y `context.lastObjectiveSlug`.

Resolucion de referencias:

- Para `asociar esa tarea al objetivo X`, usar `sessionContext.focusedEntityType = task`.
- Para `asignar ultima tarea al objetivo X`, usar foco o `context.lastTaskList` si tiene una unica tarea segura.
- Si hay ambiguedad, retornar missingData y no ejecutar.
