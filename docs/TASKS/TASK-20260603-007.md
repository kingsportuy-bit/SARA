## Problema a resolver
El sistema ya crea notas desde Chatwoot y la prueba productiva fue exitosa.
Pero el encabezado de grupo Chatwoot llega como parte del texto:

```text
**+598 91 608 727 - Fabian:**

nota: tercera prueba real de notes create con RPC cache corregida
```

Hoy esa limpieza vive temporalmente cerca de patrones/clasificadores.
Eso viola la separacion de responsabilidades.
El canal debe normalizar el contenido antes de que cualquier modulo de dominio lo vea.

Ademas, el modulo `notes` solo permite crear.
Para que sea util como MVP debe responder consultas simples:

```text
que notas tengo?
listar notas
ultimas notas
busca notas sobre foco
notas sobre foco
```

## Arquitectura esperada

### 1. Message normalizer
Crear una capa propia, por ejemplo:

```text
src/modules/messageNormalizer.ts
```

Responsabilidad:
- recibir mensajes crudos del buffer;
- preservar `originalContent`;
- producir `content` normalizado para el pipeline;
- registrar metadata minima de normalizacion si es util;
- remover encabezados de grupo Chatwoot antes de clasificadores.

Contrato sugerido:

```ts
export type NormalizedMessage = {
  id: string;
  content: string;
  originalContent: string;
  createdAt: string;
  normalization: {
    removedChatwootGroupHeader: boolean;
  };
};
```

Regla:
- `sara_messages` conserva contenido crudo.
- El worker usa contenido normalizado para clasificar/intencionar/responder.
- Los modulos de dominio no conocen formato Chatwoot.

### 2. Header Chatwoot
El normalizador debe remover al inicio el encabezado markdown de grupo.

Caso obligatorio:

```text
**+598 91 608 727 - Fabian:**

mensaje real
```

Resultado:

```text
mensaje real
```

No debe vivir en:
- `notesModule`;
- `notesStore`;
- `coarseClassifier`;
- `moduleIntentClassifier`;
- `patterns.ts`.

### 3. Notes read-only MVP
Agregar acciones:
- `notes.list`
- `notes.search`

`notes.list`:
- lista las ultimas notas por `created_at desc`;
- limite default sugerido: 5;
- maximo sugerido: 10;
- puede aceptar filtro `noteType` si se detecta de forma simple.

`notes.search`:
- busca por texto en `content`;
- limite default sugerido: 5;
- maximo sugerido: 10;
- puede aceptar filtro `noteType` si se detecta de forma simple.

Repositorio:
- agregar metodos read-only sobre `sara_notes`;
- no usar RPC de escritura;
- no crear migraciones;
- no escribir en DB para listar/buscar.
