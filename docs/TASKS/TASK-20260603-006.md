## Intencion funcional
Cuando llegue al worker un buffer con mensaje explicito de nota:

```text
nota: recordar que dormir poco me baja el foco
```

SARA debe:
1. Clasificar modulo `notes`.
2. Detectar intencion `notes.create`.
3. Validar route `notes.create`.
4. Ejecutar `notes.create` via writer/adaptador canonico.
5. Confirmar al usuario solo si existen `noteId` y `eventId`.
6. Registrar completion del buffer con la respuesta enviada.

Si el mensaje no es una accion ejecutable:
1. Mantener fallback DeepSeek.
2. No ejecutar accion.
3. No afirmar que guardo o hizo algo.

## Arquitectura esperada

### Orquestador post-buffer
Crear un modulo nuevo o evolucionar el actual de forma clara.

Nombre sugerido:
- `src/modules/bufferProcessor.ts`

Puede conservarse `createBootstrapProcessor` como wrapper de compatibilidad si simplifica el diff, pero la responsabilidad debe quedar clara:
- reclamar buffers;
- construir input versionado;
- invocar pipeline;
- enviar respuesta;
- completar o fallar buffer.

No poner logica de negocio en `chatwootIngress`.

### Flujo exacto
Por cada `ClaimedBuffer`:
1. Convertir mensajes `{ id, content, created_at }` a `{ id, content, createdAt }`.
2. Ejecutar `coarseClassifier.classify`.
3. Ejecutar `moduleIntentClassifier.classify`.
4. Ejecutar `moduleRouter.route`.
5. Si `route.executable = true`, construir `ActionExecutionInput`.
6. El `ActionExecutionInput` debe incluir:
   - `intentConfidence: intent.confidence`
   - `intentMissingData: intent.missingData`
   - `entities: intent.entities`
   - `requiresConfirmation: intent.requiresConfirmation`
7. Ejecutar `actionExecutor.execute`.
8. Ejecutar `responseComposer.compose`.
9. Enviar respuesta por Chatwoot.
10. Llamar `store.complete`.

### Fallback DeepSeek
Usar DeepSeek solo si:
- `coarse.module = "unknown"`; o
- `intent.action = "none"`; o
- `route.executable = false` y no hay accion segura para ejecutar.

No usar DeepSeek despues de una accion ejecutada, fallida, bloqueada por missingData o bloqueada por confirmation.
En esos casos responde `responseComposer`.

- `ActionExecutionResult.status = "executed"`;
