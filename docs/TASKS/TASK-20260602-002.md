# TASK-20260602-002
estado: SPEC
owner: Agente Ejecutivo
aprobador: Yo Integrado
objetivo: Implementar el primer slice vertical de ingreso Chatwoot con persistencia trazable y buffer durable de 20 segundos.
alcance_permitido: webhook Chatwoot, filtro de cuenta/inbox/conversacion autorizada, validacion de firma, idempotencia, migraciones `sara_`, buffer durable, worker de reclamo, tests y documentacion
alcance_prohibido: clasificacion LLM, ejecucion de acciones, envio de respuestas Chatwoot, frontend, cambios a objetos DB sin prefijo `sara_`, modificaciones a otras apps del VPS
validaciones: contratos, tests unitarios e integracion, guard de migraciones `sara_`, smoke local del webhook
evidencia: migraciones, tests, logs trazables y actualizacion documental

## Entregables minimos
1. Endpoint `POST /api/v1/webhooks/chatwoot`.
2. Verificacion de firma HMAC con body crudo.
3. Filtro estricto de `account=6`, `inbox=44`, `conversation=20`, evento y direccion.
4. Persistencia idempotente de entrega y mensaje.
5. Buffer durable con ventana renovable de 20 segundos.
6. Worker que reclama lotes vencidos sin doble procesamiento.
7. Tests de aceptacion, descarte, duplicados, buffer y concurrencia.

## Criterio de cierre
- Pasa a `DONE` cuando el webhook recibe, registra y agrupa mensajes validos sin invocar LLM ni tocar objetos ajenos a `sara_`.
